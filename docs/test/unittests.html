Unit Tests

Testing JSwat starts with the unit tests. These are bits of Java code that exercise the JSwat classes in order to ensure their proper functioning. Most of the classes in the core module are covered by one or more unit tests.

Preparation

Preparation for running the unit tests consists of following the instructions for compiling JSwat.

Running the Tests

Once the preparation is complete, running the tests is easy:

  1. Run the test target of the build.xml file in the module source directory.

Debugging the Tests

If you need to debug the unit tests then follow these simple steps (Note that this must be done in the NetBeans IDE to be able to invoke the debugger.):

  1. Define the test.class property in the project.properties file in the module's nbproject directory. Set the property value to the name of the unit test class.
  2. Run the debug-test-single target of the build.xml file in the module source directory.
